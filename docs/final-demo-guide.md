# 最终录屏指南

这份文档用于最终提交前录屏。目标不是把所有细节讲完，而是用 5 到 8 分钟证明：这是一个完整、可运行、可校验、可继续扩展的 AI 小说转剧本工作台。

## 录屏前检查

在录屏前先运行：

```bash
npm install
npm test
npm run lint
npm run build
npm run dev
```

浏览器打开：

```text
http://localhost:3000
```

如果本机 3000 端口被其他 Next.js 项目占用，先关闭其他项目，再启动本项目。

## 推荐演示顺序

### 1. 开场说明

说明本项目选择“AI 小说转剧本工具”，原因是：

- 输入输出稳定，适合三天内持续交付。
- YAML 剧本可以被校验、编辑和导出。
- mock provider 保证无 API Key 也能录屏。
- OpenAI-compatible provider 保留真实模型能力。

### 2. 导入小说

点击“导入文本”，选择：

```text
samples/novel-3chapters.txt
```

说明：

- 文件只在浏览器本地读取。
- 支持 `.txt` / `.md`。
- 不上传服务器。
- 文件名会生成作品标题。

### 3. 展示章节大纲

指出页面已经识别出：

- 3 个章节。
- 每章标题。
- 每章字数。
- 正文预览。

这里要强调：系统不是黑盒生成，转换前先暴露解析结果。

### 4. 展示模型配置

展示“模型配置”区域：

- 默认 `mock`，适合稳定演示。
- 可以切换到 `openai-compatible`。
- 可以填写 base URL、model、temperature、一次性 API Key。

强调：

- API Key 只随本次请求发送。
- 不写入仓库。
- 不保存到 localStorage 草稿。

### 5. 转换为 YAML 剧本

点击“转换为 YAML 剧本”。

展示右侧 YAML 包含：

- `metadata`
- `characters`
- `scenes`
- `summary`

说明这是结构化剧本初稿，不是普通文本。

### 6. 展示校验与质量清单

展示：

- Schema 校验通过。
- 剧本质量清单全部通过。
- 转换总结里的章节、角色、场景、台词数量。

说明：

- 缺字段不会用空字符串兜底。
- 坏 YAML 不能导出。
- 质量清单是结构交付检查，不是 AI 主观剧情评分。

### 7. 展示本地草稿

点击“保存为新草稿”。

刷新页面后点击“加载”，展示：

- 标题恢复。
- 小说正文恢复。
- YAML 恢复。
- 转换报告恢复。

再点击“删除”，说明删除草稿不会清空当前编辑区。

### 8. 展示失败路径

手动删除 YAML 中的：

```yaml
metadata:
  title: ...
```

展示：

- Schema 校验失败。
- 质量清单出现失败项。
- “导出 YAML”按钮不可用。

恢复字段后展示校验通过。

### 9. 导出与 Schema 文档

点击“导出 YAML”。

最后打开：

```text
docs/script-yaml-schema.md
```

说明 Schema 设计原因：让 AI 输出可编辑、可校验、可持续迭代。

## 真实 API 演示建议

如果要演示真实模型：

1. 先用 mock 完整跑通一遍。
2. 再切换到 `openai-compatible`。
3. 填写 base URL、model、API Key。
4. 如果真实模型输出未通过 Schema，直接说明：系统会拒绝坏 YAML，而不是悄悄兜底。

录屏主线不要依赖真实 API。真实 API 可以作为加分项，不要作为唯一演示路径。

## 避免录屏翻车

- 不要展示真实 API Key。
- 不要在录屏中打开无关目录或账号。
- 不要临时修改代码。
- 不要删除 `samples/novel-3chapters.txt`。
- 如果端口 3000 被占用，先关闭其他项目。
- 如果真实模型失败，切回 mock provider。

## 最终交付一句话

本项目完成了从小说文本导入、章节解析、AI 转换、YAML 校验、结构质量检查、本地草稿到 YAML 导出的完整闭环。
