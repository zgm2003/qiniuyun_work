# AI 小说转剧本工具设计

## 目标

三天内交付一个可录屏、可复现的 AI 辅助剧本创作工具：输入 3 个章节以上小说文本，输出结构化 YAML 剧本初稿，并提供 YAML Schema 文档和校验能力。

## 取舍

选择题目三，不做实时语音类题目。语音题的核心风险是麦克风、网络、ASR/TTS 延迟和评测准确度，三天内很容易做出不稳定演示。小说转剧本的输入输出稳定，适合持续提交、自动测试和录屏展示。

## 架构

- Next.js + TypeScript 单体应用。
- `src/lib` 存放纯逻辑：章节切分、剧本 Schema、YAML 校验、mock 转换器。
- `src/app/api/convert/route.ts` 只处理请求和调用转换器。
- 前端页面只负责输入、触发转换、展示 YAML、展示校验错误和导出。

## 数据结构

核心数据只有三类：

- `NovelInput`：原始小说文本和可选标题。
- `ScriptDocument`：剧本 YAML 对应的稳定结构。
- `ConversionReport`：章节数、场景数、角色数、可量化反馈。

剧本结构以 YAML 为主，不设计数据库。比赛演示不需要账号、协作、云端保存，这些复杂度先删除。

## AI 策略

默认使用 mock provider，保证没有 API Key 也能完整录屏。后续可通过环境变量接入 OpenAI 兼容接口：

- `AI_PROVIDER=mock`：本地确定性生成。
- `AI_PROVIDER=openai-compatible`：调用兼容 Chat Completions 的服务。

## 校验策略

用 Zod 定义运行时 Schema。AI 输出先 parse YAML，再 Zod 校验。校验失败不允许导出，直接显示字段路径和错误原因。字段缺失不静默兜底。

## 录屏路径

1. 打开首页。
2. 点击加载示例小说。
3. 显示章节数量校验通过。
4. 点击转换。
5. 展示 YAML 剧本和转换总结。
6. 手动删掉一个必填字段，展示 Schema 校验失败。
7. 恢复后导出 YAML。
8. 打开 Schema 文档说明设计原因。

## 不做什么

- 不做账号体系。
- 不做云端项目管理。
- 不做多用户协作。
- 不做复杂剧本行业全量格式。
- 不做不可复现的纯远程 AI 演示。
