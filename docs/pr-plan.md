# PR 切分计划

比赛要求持续 PR 和 commit，不能最后一天一次性导入。建议按以下顺序开 PR：

1. **初始化项目骨架**：Next.js、TypeScript、README、环境变量示例。
2. **章节切分能力**：识别 3 章以上小说输入，给出明确错误。
3. **YAML Schema 能力**：定义剧本结构、校验器和 Schema 文档。
4. **剧本转换核心**：mock provider 和示例输入输出。
5. **交互界面**：输入、转换、编辑、校验、导出。
6. **演示文档**：录屏脚本、依赖说明、原创功能说明。

每个 PR 描述必须包含：功能描述、实现思路、测试方式。
